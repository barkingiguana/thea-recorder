Server & API Reference REST API for display, panels, and recording control

The thea-recorder server is a long-running process that wraps a single recorder instance and exposes it via a REST API. SDKs and the CLI both talk to this server.

Starting the server

thea serve --port 9123 --output-dir ./recordings --display 99bash

Server options

FlagDefaultDescription
--port9123Port to listen on
--display99X11 display number for Xvfb
--output-dir/tmp/recordingsWhere MP4 files are saved
--browser-size1920x1080Virtual display resolution
--framerate15Recording FPS
--corsoffEnable CORS headers for browser access

API Reference

All endpoints return JSON. Error responses use {"error": "message"}.

Display

POST /display/start

Start the virtual display (Xvfb).

curl -X POST http://localhost:9123/display/start
Response 201
{"status": "started", "display": ":99"}

Returns 409 if display is already started.

POST /display/stop

Stop the virtual display.

curl -X POST http://localhost:9123/display/stop
Response 200
{"status": "stopped"}

Panels

POST /panels

Create a panel overlay. width is optional (null = auto-width). title defaults to empty string.

curl -X POST http://localhost:9123/panels \
  -H "Content-Type: application/json" \
  -d '{"name": "status", "title": "Status", "width": 120}'
Response 201
{"name": "status", "title": "Status", "width": 120}

Returns 400 if name is missing or width is invalid.

GET /panels

List all panels.

curl http://localhost:9123/panels
Response 200
[{"name": "status", "title": "Status", "width": 120}]
PUT /panels/:name

Update panel content. focus_line defaults to -1 (focus on last line).

curl -X PUT http://localhost:9123/panels/status \
  -H "Content-Type: application/json" \
  -d '{"text": "Running step 3", "focus_line": 5}'
Response 200
{"name": "status", "text": "Running step 3"}

Returns 404 if panel does not exist.

DELETE /panels/:name

Remove a panel.

curl -X DELETE http://localhost:9123/panels/status
Response 200
{"status": "removed"}

Returns 404 if panel does not exist.

Recording

POST /recording/start

Begin recording. Requires a name field.

curl -X POST http://localhost:9123/recording/start \
  -H "Content-Type: application/json" \
  -d '{"name": "login_test"}'
Response 201
{"status": "recording", "name": "login_test"}

Returns 409 if already recording. 400 if name is missing.

POST /recording/stop

Stop recording. Returns the file path and elapsed time.

curl -X POST http://localhost:9123/recording/stop
Response 200
{"path": "/app/recordings/login_test.mp4", "elapsed": 45.2, "name": "login_test"}

Returns 409 if not currently recording.

GET /recording/elapsed

Get elapsed recording time in seconds. Returns 0.0 when not recording.

curl http://localhost:9123/recording/elapsed
Response 200
{"elapsed": 12.5}
GET /recording/status

Get current recording state.

curl http://localhost:9123/recording/status
Response 200
{"recording": true, "name": "login_test", "elapsed": 12.5}

Recordings (file access)

GET /recordings

List all saved recordings.

curl http://localhost:9123/recordings
Response 200
[
  {
    "name": "login_test",
    "path": "/app/recordings/login_test.mp4",
    "size": 1234567,
    "created": "2026-03-10T14:30:00+00:00"
  }
]
GET /recordings/:name

Download a recording as MP4. Supports range requests for video seeking.

# Download
curl -O http://localhost:9123/recordings/login_test

# Range request
curl -H "Range: bytes=0-1023" http://localhost:9123/recordings/login_test

# wget
wget http://localhost:9123/recordings/login_test -O login_test.mp4

Returns 404 if recording does not exist. 400 if name contains path traversal characters. Range requests return 206 Partial Content.

GET /recordings/:name/info

Get metadata for a recording without downloading it.

curl http://localhost:9123/recordings/login_test/info
Response 200
{
  "name": "login_test",
  "path": "/app/recordings/login_test.mp4",
  "size": 1234567,
  "created": "2026-03-10T14:30:00+00:00"
}

Health & Utility

GET /health

Server health check. Returns display state, active panels, and uptime.

curl http://localhost:9123/health
Response 200
{
  "status": "ok",
  "recording": false,
  "display": ":99",
  "panels": ["status", "scenario"],
  "uptime": 123.4
}
POST /cleanup

Full teardown: stops any active recording, stops the display, removes all panels.

curl -X POST http://localhost:9123/cleanup
Response 200
{"status": "cleaned"}

Error Responses

All errors return JSON with an error field:

{"error": "panel 'foo' not found"}
StatusMeaning
400Bad request (missing/invalid fields)
404Resource not found (panel or recording)
405Method not allowed
409Conflict (display already started, already recording, not recording)
416Range not satisfiable

CORS

Enable CORS with --cors to allow browser-based dashboards to access recordings:

thea serve --port 9123 --cors

This adds Access-Control-Allow-Origin: * to all responses and handles preflight OPTIONS requests.

Graceful Shutdown

The server handles SIGTERM gracefully:

  1. Stops any in-progress ffmpeg recording (video is finalised)
  2. Stops Xvfb
  3. Removes panel temp files
  4. Exits cleanly
Docker safe This means docker stop will not lose a recording in progress.

Thread Safety

All endpoints that mutate recorder state are protected by a threading.Lock. Concurrent callers (e.g., multiple SDK clients or panel updates from different threads) are safe.
