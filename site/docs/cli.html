CLI Reference Server and client commands

The thea command is automatically available after pip install thea-recorder. It has two modes: server mode and client mode.

Two modes

ModeWhat it does
Server modethea serve starts a long-running HTTP server
Client modeAll other subcommands send HTTP requests to a running server

Server mode

thea serve --port 9123 --output-dir ./recordingsbash

See Server & API Reference for all server options and the full REST API.

Client mode

Every client command talks to a running server via HTTP.

Server URL configuration

MethodExample
--server flagthea --server http://host:9123 health
RECORDER_URL envexport RECORDER_URL=http://host:9123
Defaulthttp://localhost:9123

Output flags

FlagEffect
(default)JSON output, one line
--prettyIndented JSON
--quietNo output (exit code only)

Non-zero exit code on any error.

Display

# Start the virtual display
thea start-display

# Stop the virtual display
thea stop-displaybash

Panels

# Add a panel
thea add-panel --name status --title Status --width 120

# Update panel content
thea update-panel --name status --text "Running step 3" --focus-line 5

# Remove a panel
thea remove-panel --name statusbash

Recording

# Start recording
thea start-recording --name login_test

# Check elapsed time
thea elapsed

# Stop recording (prints path and elapsed)
thea stop-recordingbash

File operations

# List all recordings
thea list-recordings

# Download a recording
thea download --name login_test --output ./login_test.mp4bash

Utility

# Health check
thea health

# Full teardown
thea cleanup

# Version
thea versionbash

Scripting examples

Record a test from bash

#!/bin/bash
set -euo pipefail

export RECORDER_URL=http://localhost:9123

thea start-display
thea add-panel --name status --title Status --width 120
thea start-recording --name "my_test"
thea update-panel --name status --text "Running"

# ... run your test here ...

thea update-panel --name status --text "PASSED"
RESULT=$(thea stop-recording)
echo "Video: $(echo $RESULT | jq -r .path)"

thea cleanupbash

List recordings as a table

thea list-recordings --pretty | jq -r '.[] | "\(.name)\t\(.size)\t\(.created)"'bash

Pipeline: record + download

thea start-recording --name smoke_test
sleep 10  # run tests
thea stop-recording --quiet
thea download --name smoke_test --output ./smoke_test.mp4bash

Error messages

When the server is unreachable:

Error: cannot reach recorder server at http://localhost:9123 — is 'thea serve' running?

When a resource does not exist:

Error: panel 'foo' not found

When there is a conflict:

Error: already recording
