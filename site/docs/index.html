Getting Started Record your E2E tests. Watch them back.

thea-recorder captures browser tests as MP4 video with live panel overlays. Run the server on any Linux machine with a virtual display, then control it from any language over HTTP.

Good to know System dependencies (Xvfb, ffmpeg) are only needed on the machine running the server. Your test code just talks to it over HTTP — install the SDK for your language and go.

Install the server

# Install system deps (Linux)
sudo apt-get install xvfb ffmpeg

# Install thea-recorder
pip install thea-recorder

Start the server

thea serve

The server is now listening on http://localhost:9123. It manages the virtual display, panel overlays, and ffmpeg recording.

Pick your language and record

from thea import RecorderClient

client = RecorderClient("http://localhost:9123")
with client.recording("login_test"):
    ...  # your test code herepython

The thea CLI is included with pip install thea-recorder — CLI docs

# Start display
curl -X POST http://localhost:9123/display/start

# Start recording
curl -X POST http://localhost:9123/recording/start \
  -H "Content-Type: application/json" \
  -d '{"name": "login_test"}'

# Stop recording
curl -X POST http://localhost:9123/recording/stopcurl
